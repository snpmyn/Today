Android SDK 版本说明

1. compileSdkVersion

编译应用时使用的 SDK 版本，决定可用 API，仅影响编译，不影响安装。

compileSdkVersion 34 // 使用 Android 14 编译

2. minSdkVersion

应用可安装的最低系统版本（API 级别）。低于此版本的设备无法安装。

minSdkVersion 24 // Android 7.0 及以上可安装

3. targetSdkVersion

声明应用针对哪个版本测试过，系统会基于此启用或关闭行为限制。

targetSdkVersion 34 // 针对 Android 14 测试

总结对比

属性 作用 影响安装 示例
compileSdkVersion 编译时可用 API 否 34 (Android 14)
minSdkVersion 最低安装系统 是 24 (Android 7.0+)
targetSdkVersion 行为适配 否 34 (Android 14)

SDK 版本关系示意

minSdkVersion (最低安装)
targetSdkVersion (行为适配)
compileSdkVersion (编译 API)

⬆️ 从下到上理解：编译 → 适配 → 安装门槛

选择流程

1️⃣ minSdkVersion
决定最低可支持系统
2️⃣ targetSdkVersion
选择你测试过或最新系统
3️⃣ compileSdkVersion
通常与 target 一致或更高

💡 建议：minSdkVersion 保持兼容性，target 最新保证适配，compile 最新保证编译可用 API。
